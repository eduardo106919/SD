## Grupo 01

### Questão 01

Num sistema orientado a objetos, a invocação remota visa tornar a rede "transparente", fazendo com que uma chamada a um objeto noutra máquina pareça uma chamada local.

Componentes e Objetivos:
- Stub no Cliente: Atua como um representante local do objeto remoto. O seu objetivo é empacotar os argumentos e enviar o pedido pela rede.
- Dispatcher: Recebe o pedido no servidor e identifica qual o objeto e método específicos que devem ser executados, encaminhando a chamada.
- Skeleton no Servidor: Faz o processo inverso do Stub. Ele descompacta os argumentos, chama o método real no objeto servidor e empacota o resultado para enviar de volta.
- Módulo de Comunicação: Responsável pelo transporte físico dos bytes entre as máquinas, gerindo protocolos e conexões.
- Remote Reference Modules: Mantêm uma tabela para traduzir referências de objetos locais em referências remotas globais, garantindo que o cliente comunica com a instância correta no servidor.


### Questão 02

A migração de código envolve a transferência de programas executáveis entre nós de um sistema distribuído. Os principais problemas que resolve são:
- **Latência de Rede**: Ao mover o código para junto dos dados, evita-se o envio constante de grandes volumes de informação pela rede para serem processados num servidor central, reduzindo o tempo de resposta.
- **Escalabilidade e Carga do Servidor**: Permite descarregar o processamento do servidor para os clientes (como scripts executados no browser), distribuindo o esforço computacional e evitando que o servidor se torne um gargalo.
- **Largura de Banda**: Em sistemas com ligações lentas, é mais eficiente enviar o "algoritmo" (pequeno em tamanho) para processar dados localmente do que transferir os dados brutos.
- **Adaptabilidade**: Permite que o sistema mova serviços para nós com mais recursos disponíveis em tempo real, melhorando o balanceamento de carga.


### Questão 03

A Transparência de Localização é a propriedade que permite ao utilizador ou aplicação aceder a um recurso (como um ficheiro ou objeto) sem saber onde ele está fisicamente localizado na rede.
- **Definição**: O nome do recurso não deve conter qualquer informação sobre o endereço da máquina onde ele reside.
- **Método Genérico** para a atingir: O método padrão é o uso de uma Indireção através de um Serviço de Nomes ou um sistema de diretoria.
    - Em vez de usar um endereço físico (IP, Porta), o sistema utiliza um Nome Lógico estável.
    - O sistema consulta uma tabela de mapeamento (o serviço de nomes) que traduz esse nome lógico no endereço físico atual do recurso.
    - Isto permite que o recurso mude de servidor sem que o cliente tenha de alterar o seu código ou forma de acesso.
